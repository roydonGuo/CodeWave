完成作品分类需求，再作品页有作品分类集合展示，作品分类属性为{name，id}。
从后端接口：/cw/postCategory/myList 查询接口列表，为get方法，请求参数使用pageNum=1和pageSize=99
接口响应数据为格式为：
{
    "total": 1,
    "rows": [
        {
            "id": 2,
            "name": "Java"
        }
    ],
    "code": 200,
    "msg": "查询成功"
}

作品需求列表要在首节点加上”全部“这个作品分类，因为后续需要针对于分类查询作品列表，”全部“这个分类的id设置为空，且是默认选择的。
目前项目种作品分类是定义死的数据，现要替换为从后端接口获取。

且要完成一个作品分类添加弹窗，在作品页面合适位置添加一个新增分类按钮，点击后弹出添加作品分类弹窗。
